# 줍줍스2 (v2)

줍줍스1(`src/features/feature-1/`)을 포크해 **조작 피드백(게임필)** 과 **모바일 품질**에 집중해 개선한 버전.
`/features/1/v2` 에서 플레이하고, 진행 데이터는 v1과 완전히 분리된다.

## v1과의 차이

| 축 | 줍줍스1 | 줍줍스2 |
|---|---|---|
| **타격감** | 없음 | **히트스톱**(등급별 25~90ms, 위성 충돌 140ms) — 충격 순간 시뮬을 멈추고 렌더만 돌린다 |
| **연속 흡수** | 개별 XP | **콤보 체인** 4단계(GOOD→KESSLER!, ×1.25~×3). 콤보가 오를수록 유지 창이 좁아진다 |
| **위험** | 충돌은 손해일 뿐 | **니어미스** — 위성을 아슬아슬하게 스치면 슬로모 + XP. 피하기가 능동적 플레이가 된다 |
| **화면 흔들림** | 랜덤 지터 | **트라우마 모델** — 제곱 감쇠 + value noise. 약한 충격은 안 흔들리고 큰 충격만 확 온다 |
| **사운드** | 없음 | **절차적 WebAudio** (에셋 0바이트). 콤보가 오르면 펜타토닉 사다리로 음이 상승 |
| **햅틱** | 없음 | `navigator.vibrate` — 흡수·충돌·진화·콤보 상승 |
| **조작** | 절대좌표 추적 → **손가락이 줍스를 가림** | **상대 드래그** — 터치는 손가락 위 52px로 띄운다. 스프링 물리로 관성이 붙는다 |
| **성능** | 스프라이트마다 최대 256 fillRect | **오프스크린 베이크 + drawImage 1회**, 파티클 풀(할당 0), 품질 티어 자동 강등 |
| **세이프에어리어** | 미대응 | 라우트 `viewport` export로 `viewportFit: cover` + `env(safe-area-inset-*)` |
| **절전** | 백그라운드에서도 rAF | `visibilitychange`에 rAF 정지 + 오디오 서스펜드 |
| **리텐션** | 부재 보고서 | + **일일 미션 3종**(자정 리셋) + **접속 스트릭**(XP ×1.5까지, 3일마다 글로벌 링크 코어) |

## 핵심 설계 노트

**게임 클록(`gt`)이 히트스톱의 전제 조건.** v1은 모든 스케줄이 `performance.now()` 벽시계 기준이었다.
그대로 히트스톱을 넣으면 정지 중에도 벽시계는 흘러서, 정지가 풀리는 순간 밀린 스폰이 한꺼번에 터진다.
v2는 내부 시각 `gt`만 쓰고 `frame()`에서만 벽시계를 받는다. 덕분에 슬로모(`timeScale`)와
백그라운드 복귀 시 dt 폭주 방지도 공짜로 따라온다.

**콤보 HUD는 캔버스에 그린다.** React 상태로 올리면 프레임마다 리렌더가 일어나 모바일에서 60fps가 무너진다.
캔버스에 직접 그리고, React에는 250ms 스로틀로만 올린다.

  진동(안드로이드 크롬만), iOS 오디오 언락(첫 탭 전에는 무음이 정상)
