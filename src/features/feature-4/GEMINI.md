## 프로젝트 개요

- **스택**: Next.js 15.5(App Router, Turbopack) + React 19 + Tailwind CSS 4. 신규 npm 의존성 없이 표준 웹 API(Canvas 2D, Fullscreen API, localStorage)만으로 구현.
- **이 기능의 정체성**: 메인 화면 9개 버튼 중 4번 "줍스 오비트" — 우주쓰레기/케슬러 신드롬을 소재로 한 애완 생명체 육성 + 궤도 청소 게임.
- **진입점**: `/features/4` (관제소), `/features/4/play` (궤도 비행 미니게임), `/features/4/tracking` (관제 센터). 메인 화면 버튼 문구는 `config.ts`에서만 관리(자동 반영).

## 게임 컨셉 — 사용자 원본 요구사항 (변경 없이 보존)

이 게임의 세계관/기획 의도이며, 이후 기능 추가 시 이 방향성을 벗어나면 안 됩니다.

1. **배경 소재**: 우주쓰레기, 케슬러 신드롬을 배경에 둔 우주 게임.
2. **비주얼 톤**: 영화 《그래비티》처럼 신비적이고 몽환적인 지구를 배경으로 한다.
   - **2-1. 로드맵(소셜)**: 차후 전세계 유저가 자신의 쓰레기 청소 성과를 자랑·공유하고, 순위(랭킹)가 있고, 자기 펫을 자랑하는 방향으로 발전시킨다. → 현재는 UI 스텁만 존재(§구현 현황 참고), 백엔드 없음.
   - **2-2. 로드맵(세계관 확장)**: 지금은 지구가 배경이지만 차후 화성, 목성 …으로 세계관을 확장한다. → 현재는 `constants.ts`의 `WORLDS` 배열에 잠금 상태로만 정의됨, 콘텐츠 없음.
   - **2-3. 톤 대비**: 배경은 몽환적이고 우주 덕후들이 좋아할 요소(궤도, 위성, 오로라, 대기 등)로 가득하되, 게임 캐릭터는 두들스럽고 귀엽고 뽀짝한 느낌을 유지한다. 이 대비 자체가 매력 포인트.
3. **캐릭터 설정**: "줍스"는 케슬러 신드롬 극복을 위한 세계 과학자들의 시도로 탄생한, 우주 환경에서 살아가며 우주쓰레기를 먹고 그걸로 추진력·체력을 얻고 스킬·진화를 하는 애완 생명체.
4. **애완성을 부여한 이유(설계 근거)**: 우주는 너무 넓고 쓰레기는 너무 작아서, 지구 시민 모두가 한 마리 이상의 줍스와 상호작용하며 보살피고 컨트롤 훈련시켜야만 청소 목표를 달성할 수 있다는 세계관적 필연성. → 다마고치형 돌봄 메커닉의 존재 이유.
5. **쓰레기 다양성 & 리스크**: 다양한 크기·물성의 우주쓰레기가 있고, 진화·아이템·스킬 정도에 따라 처리 가능한 쓰레기가 달라진다. 인공위성이나 처리 못 하는 물체와 충돌하면 체력이 고갈되며, 이때는 다마고치처럼 주인(플레이어)의 보살핌이 필요하다.
6. **지구 표현**: 그래비티 영화 배경처럼 신비스럽되, 대륙 형태가 표현되어 궤도 비행 중 지구가 회전하는 것처럼 느껴져야 한다. 사실적인 지도·구름·바다 — **지도는 대한민국 정도 크기가 식별되어야 한다.**
7. **줍스 간 조우**: 다른 줍스들을 만나는 경우가 있고, 만나면 경험치가 증가하며 깜찍한 효과가 있어야 한다.
8. **영속성**: 플레이는 영속적이다. 사용자가 플레이하지 않아도 줍스는 지구 궤도를 돌며 계속 청소하고 있다.
9. **관제 메뉴**: 사용자는 줍스가 지금 어느 위치에서 어느 궤도를 도는지 확인 가능한 메뉴가 있다. NASA 인공위성 관제 화면처럼 지구본을 펼친 화면에 궤도·위성 위치·어느 나라 상공인지·주인 머리 위 도달 시각을 보여준다.
10. **진화·아이템에 의한 확장**: 진화·스킬에 따라 주인 상공 교신 범위가 증가하고, 일시적 아이템을 먹으면 전지구적으로 교신 가능해진다. 이 시점엔 주인이 다마고치처럼 보살필 수 있고, 플레이하면 더 많은 경험치·아이템을 받는다.

### 화면 3개 (요구사항 대응)

| 화면 | 라우트 | 담당 컴포넌트 | 대응 요구사항 |
|---|---|---|---|
| 관제소(Hub) | `/features/4` | `HubScreen.tsx` (+ `Onboarding.tsx`) | 1, 3, 4, 5(돌봄), 8, 10(교신), 2-1(랭킹/공유 스텁), 2-2(로드맵 스텁) |
| 궤도 비행(Play) | `/features/4/play` | `PlayGame.tsx` | 2, 2-3, 3, 5, 6, 7 — 캔버스 2D 미니게임 |
| 관제 센터(Tracking) | `/features/4/tracking` | `TrackingScreen.tsx` | 6, 9 — NASA 관제 화면풍 지도 |

첫 방문 시 `Onboarding.tsx`가 3장 세계관 슬라이드 + 줍스 이름 짓기를 진행하고, 이후 `useGameState` 훅이 매 5초 및 재방문 시 오프라인 시뮬레이션을 자동 적용한다(요구 8: 접속 안 해도 청소 진행).

### 라이브러리 계층 (`lib/`)

- **`types.ts`**: `SaveState`(영속 저장 스키마), `StageDef`(진화 단계), `DebrisType`, `OrbitParams` 등 전체 타입.
- **`constants.ts`**: 진화 5단계(`STAGES`), 쓰레기 6종 도감(`DEBRIS`), 궤도 상수(ISS 유사: 고도 420km, 경사각 51.6°, 주기 92분48초), `WORLDS`(지구/화성/목성 로드맵 잠금 정의).
- **`state.ts`**: localStorage 저장/로드, **`applyOffline()`**— 마지막 접속 이후 경과 시간을 10분 청크로 시뮬레이션(자율 청소, 확률적 충돌·친구 조우, XP·레벨업)하는 영속성 엔진의 핵심. 돌봄 액션(먹이·쓰다듬기·회복·휴식·부스터)도 여기 있음.
- **`orbit.ts`**: 원궤도 지상궤적 계산(`groundPoint`), 대권거리(`greatCircleKm`), 교신 창 탐색(`nextContact`/`contactWindows`), 태양 직하점(주야 경계 계산용).
- **`geo.ts`**: `public/feature-4/world-50m.json`(Natural Earth 50m을 Douglas-Peucker로 경량화, 194개국·~346KB) 로딩 + point-in-polygon 판정으로 "지금 어느 나라 상공인가" 계산, 영문→한글 국가명 매핑.
- **`earthTexture.ts`**: 대륙 데이터로 지구 텍스처를 오프스크린 캔버스에 렌더. `buildEarthTexture`(낮/관제 스타일), `buildNightTexture`(도시 불빛 ~1,600개, 해안선 앵커 샘플링), `buildCloudTexture`.
- **`joopsArt.ts`**: 줍스 캐릭터를 SVG 문자열로 생성(`joopsSvg`/`joopsDataUrl`). 진화 단계(`stage`)·기분(`mood`)·hue에 따라 외형이 변한다. `<img>`와 캔버스 양쪽에서 재사용.
- **`cinematic.ts`**: (play-cinematic 브랜치에서 추가) 태양·달·오로라·렌즈플레어·비네트·필름그레인·대기 링·밝은 별 글린트 등 프리렌더 캔버스 유틸. 전부 "빌드 1회 + 프레임당 drawImage" 패턴.
- **`rng.ts`**: mulberry32 시드 난수(텍스처 재현성용).

### 훅 (`hooks/`)

- **`useGameState.ts`**: 세이브 로드 + 마운트 시 오프라인 시뮬레이션 적용 + 5초 틱 + 액션 디스패치(`mutate`) + 신규 줍스 생성(`create`).
- **`useFullscreen.ts`**: (play-cinematic 브랜치에서 추가) 표준 Fullscreen API + webkit 접두사 래핑. `document.fullscreenEnabled`로 지원 여부 판정 → 미지원 브라우저에선 버튼 자체를 숨기는 패턴.

## 이 폴더 한정 코딩 컨벤션

- **주석은 한국어, WHY만**: 코드가 무엇을 하는지 설명하는 주석 금지. 숨은 제약·비직관적 트레이드오프에만 짧게.
- **Tailwind 유틸리티만**: `globals.css` 수정 금지(공유 파일). 전역 CSS가 필요하면 `src/features/feature-4/*.module.css`.
- **다크모드 지원**, 메인 화면과 같은 톤(rounded-2xl, 검정/흰색 중립 팔레트) 유지. Play 화면은 예외적으로 우주 배경 자체가 다크 테마 고정(게임 특성상).
- **Canvas 2D 렌더링 패턴**: `build*`/`make*` 함수는 오프스크린 캔버스에 1회 프리렌더, `draw*` 함수는 매 프레임 `drawImage`만 호출. 그라데이션·패스 재계산을 루프 안에서 직접 하지 않는다(성능).
- **성능 최적화 패턴**(`PlayGame.tsx` 참고): 프레임타임 이동평균으로 저사양 기기 자동 감지(`lowFx`) → 오로라·그레인 같은 장식 효과 생략, 배경(그라데이션+성운+태양+원경별)을 정적 이미지 1장으로 프리컴포짓, 야간 지구 블렌드는 N프레임마다만 재계산(스크롤 오프셋만 보정해 블릿).
- **safe-area 처리**: 노치/홈 인디케이터를 피해야 하는 절대 위치 HUD 요소는 인라인 `style`로 `max(기본값, env(safe-area-inset-*))` 패턴 사용(Tailwind 임의값 대신 인라인 style을 쓴 이유: `env()`를 산술 `max()`와 결합해야 해서). `viewport-fit=cover`는 `src/app/features/4/play/page.tsx`의 세그먼트 단위 `viewport` export로만 적용 — 공유 루트 `layout.tsx`는 건드리지 않는다.
- **캐릭터 아트**: 새 표정/장신구 추가 시 `joopsArt.ts`의 `joopsSvg()` 한 곳만 수정하면 온보딩/허브/플레이/트래킹 전체에 반영됨(단일 소스).

## 알려진 제약 / 완료되지 않은 것

- **iOS Safari 전체화면 불가**: iPhone Safari는 일반 탭에서 `<div>` 등 임의 요소의 Fullscreen API를 지원하지 않는다(Apple 플랫폼 제약, `<video>`만 예외). JS로 우회 불가능. 현재 `useFullscreen`은 `document.fullscreenEnabled`로 이를 감지해 버튼을 숨기는 정도로 대응. 진짜 아이폰 전체화면(크롬 없는 standalone)을 원하면 **"홈 화면에 추가" PWA 설치**가 유일한 방법 — manifest, apple-touch-icon 이미지 에셋이 필요한 별도 작업이며 아직 미구현.
- **글로벌 랭킹(요구 2-1)**: `HubScreen.tsx`에 "시즌 0 준비 중" UI 스텁만 존재. 실제 서버/DB/유저 인증 없음. 공유 버튼은 `navigator.share`/클립보드로 텍스트만 복사.
- **화성·목성 확장(요구 2-2)**: `constants.ts`의 `WORLDS` 배열에 `unlockLevel`과 `live: false`만 정의되어 있고 실제 콘텐츠(맵, 쓰레기, 궤도)는 없음.
- **친구 줍스(요구 7)**: 현재는 Play 미니게임 내에서만 조우 가능(스폰 랜덤 이벤트). 허브 화면에서 오프라인 시뮬레이션 중에도 조우가 발생하지만(`state.ts`의 `applyOffline`), "내 펫 자랑" 같은 소셜 기능과는 아직 연결 안 됨.
- **테스트 자동화 부재**: 이 프로젝트엔 Playwright/Puppeteer가 설치돼 있지 않다. 브라우저 동작 검증은 시스템에 있는 `/usr/bin/google-chrome`을 `--headless=new --remote-debugging-port=...`로 띄우고 CDP WebSocket을 직접 열어(Runtime.evaluate, Page.captureScreenshot, Input.dispatchMouseEvent 등) 스크린샷·콘솔에러·FPS·실제 클릭 왕복까지 검증하는 방식을 이번 세션에서 사용했다(임시 스크립트, 저장소에는 커밋 안 함). 새 브라우저 자동화가 필요하면 이 패턴을 재사용하거나, 사용자에게 Playwright 설치 승인을 요청하라(신규 의존성 규칙).

1. `npm run lint && npm run build` 통과
2. 개발 서버: `npm run dev -- --port 3004` (포트 컨벤션 준수)
